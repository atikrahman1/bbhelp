Fuzzer Configuration

Enter one file path per line. The scanner will check these files on every page load when enabled.

Supported Variables:
• {DOMAIN} - Domain without www (e.g., example.com)
• {HOST} - Full hostname (e.g., www.example.com)
• {PROTOCOL} - http or https
• {ORIGIN} - Protocol + host (e.g., https://example.com)

Examples:
• {DOMAIN}.zip → example.com.zip
• backup-{DOMAIN}.sql → backup-example.com.sql
• {DOMAIN}-db-backup.tar.gz → example.com-db-backup.tar.gz

How long to wait before rescanning the same website (1-168 hours)

What is False Positive Protection?
When enabled: Uses advanced filtering (baseline comparison, content-type validation, duplicate detection) to reduce false positives.
When disabled: Only relies on HTTP 200 status code - shows all files that return 200 (may include catch-all pages).

Notification Control:
When enabled: Shows a popup notification when sensitive files are discovered.
When disabled: Files are still found and badge is updated, but no popup notification appears.

Sequential: Scans files one by one. Slower but stealthier, less likely to trigger WAF.
Parallel: Scans multiple files at once in batches. Much faster but more aggressive.

Number of simultaneous requests when using parallel engine (2-20)

Delay between requests in milliseconds (0-5000). Acts as rate limiting to avoid WAF/IP bans.
0ms = No delay (fastest, risky) | 100ms = Default | 500-2000ms = Stealth mode
